 Projeto de Gerenciamento de Riscos em Desenvolvimento de Software

## Descrição do Projeto

Este repositório apresenta um estudo de gerenciamento de riscos aplicado ao desenvolvimento de um aplicativo móvel para agendamento de consultas médicas. O sistema contempla funcionalidades como cadastro de usuários, gestão de agenda médica, agendamento de consultas, notificações e integração com um sistema externo de prontuário eletrônico.

O projeto encontra-se em fase intermediária de desenvolvimento e enfrenta desafios relacionados à integração com sistemas externos, mudanças de requisitos e limitações de recursos da equipe. Com base nesse cenário, foram aplicadas técnicas de identificação, análise e tratamento de riscos, além da elaboração de uma comunicação executiva para stakeholders.

## Objetivo do Exercício

O objetivo deste exercício é aplicar conceitos de gerenciamento de riscos em projetos de software, utilizando um modelo de linguagem (LLM) como apoio para:

- Identificar riscos relevantes do projeto;
- Avaliar qualitativamente os riscos identificados;
- Definir estratégias de resposta adequadas;
- Elaborar uma comunicação clara e objetiva para stakeholders;
- Demonstrar a aplicação prática das boas práticas de gestão de projetos.

## Organização do Repositório

O repositório está organizado nos seguintes arquivos:

- `README.md` → Apresentação do projeto, objetivos e estrutura do repositório.
- `Etapa_1_Identificacao_de_Riscos.md` → Lista dos riscos identificados, descrições e contexto.
- `Etapa_2_Analise_de_Riscos.md` → Análise qualitativa dos riscos, impactos e fatores condicionantes.
- `Etapa_3_Estrategias_de_Resposta.md` → Estratégias de tratamento dos riscos e ações recomendadas.
- `Etapa_4_Comunicacao_Stakeholders.md` → Comunicação executiva destinada aos stakeholders do projeto.
